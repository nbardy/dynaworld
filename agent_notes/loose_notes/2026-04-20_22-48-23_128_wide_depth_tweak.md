# 128 Wide-Depth Tweak

The raw 128px/4fps run now stays finite, but metrics showed late windows are
still geometrically broken. Examples from the user run:

```text
frames [39,40,41,42]:
FrontGaussiansMin=33 or 0 in later windows
NearOrBehindGaussiansMean=450-510
NoContributionSlotFraction around 0.8-0.99
```

This explains full-white frames and weak convergence. The renderer guard fixed
NaNs, but the old Gaussian head z range still caps known-camera TokenGS at:

```text
z in [0.5, 2.5]
```

The 128 DUSt3R camera path can move to about that same depth or beyond in the
first-camera coordinate frame. For late frames this can make it impossible for
the frame-conditioned head to place enough splats in front of the camera.

## Code Change

Made the known-camera TokenGS Gaussian head range configurable while preserving
old defaults:

```text
xy_extent default 1.5
z_min default 0.5
z_max default 2.5
scale_init default 0.05
```

New configs:

```text
src/train_configs/local_mac_overfit_prebaked_camera_128_4fps_wide_depth_8frames.jsonc
src/train_configs/local_mac_overfit_prebaked_camera_128_4fps_wide_depth_1024splats.jsonc
```

Both use:

```text
z in [0.5, 5.0]
```

The first keeps 512 splats and uses `frames_per_step=8`.
The second uses 1024 splats and keeps `frames_per_step=4`.

Do not combine both knobs yet on MPS unless we are ready for much higher dense
render memory.

## Quick Check

Forced late-window `[39,40,41,42]` at initialization:

```text
raw 128:
FrontGaussiansMin=14
NearOrBehindGaussiansMean=475.5
ContributingGaussiansMean=24.25
NoContributionSlotFraction=0.8672

wide-depth 128:
FrontGaussiansMin=439
NearOrBehindGaussiansMean=41
ContributingGaussiansMean=137
NoContributionSlotFraction=0.2695
```

One-step smokes:

```text
wide_depth_8frames:
passed, finite, debug step about 9.15s with metrics every step

wide_depth_1024splats:
passed, finite, debug step about 8.57s with metrics every step
```

Interpretation: the z range was a real representational blocker. More splats
helps detail, more frames helps temporal coverage, but the first necessary tweak
was widening depth or normalizing camera scale.
