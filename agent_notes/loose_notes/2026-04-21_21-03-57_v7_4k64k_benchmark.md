# v7 4K 64K benchmark

## Context

The user challenged the phrase "current level of maturity" for v7 and asked to
run big splats. I ran a bounded 4K/65,536 splat benchmark comparing v7 hardware
against v6 direct.

Command:

```bash
cd third_party/fast-mac-gsplat
python benchmarks/benchmark_full_matrix.py \
  --resolutions 4096x4096 \
  --splats 65536 \
  --batch-sizes 1 \
  --distributions microbench_uniform_random,clustered_hot_tiles \
  --renderers v6_direct,v7_hardware \
  --modes forward,forward_backward \
  --warmup 1 \
  --iters 2 \
  --timeout-sec 240 \
  --output-md benchmarks/full_rasterizer_benchmark_v7_4k64k.md \
  --output-jsonl benchmarks/full_rasterizer_benchmark_v7_4k64k.jsonl
```

All 8 cells completed with `status=ok`.

## Results

| Distribution | Mode | v6 direct | v7 hardware | Result |
|---|---|---:|---:|---|
| uniform random | forward | 31.301 ms | 149.561 ms | v7 slower by 377.8% |
| uniform random | forward+backward | 68.039 ms | 21574.243 ms | v7 not viable |
| clustered hot tiles | forward | 253.653 ms | 129.820 ms | v7 faster by 48.8% |
| clustered hot tiles | forward+backward | 178.227 ms | 20054.494 ms | v7 not viable |

## Interpretation

The hardware render-pipeline idea is real enough to win a clustered
forward-only 4K case. That is the most encouraging v7 result so far.

The training story is still bad. v7 forward+backward spends about 20 seconds in
the backward component at 4K/64K, while v6 direct is tens to hundreds of
milliseconds depending on distribution. This is not a tuning problem at the
margin; the current v7 architecture is not a training renderer.

## Current Model

v7 should remain a profiling branch for hardware blending and render-pipeline
experiments. It should not be promoted as a default until:

- MPS tensor to CPU/shared-buffer round trips are removed
- render textures are not read back through Torch in the hot path
- backward is redesigned rather than replaying through the current slow bridge
- 4K/64K forward+backward beats v6 on at least one representative training case
